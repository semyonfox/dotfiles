# Waybar Configuration

Dynamically generated by HyDE with Catppuccin Mocha theme.

This is the Waybar status bar configuration for Hyprland, generated by the HyDE framework's wbarconfgen.sh script. It's maintained as a stow package for portability and version control.

## Deployment

### Prerequisites

Ensure stow is installed:
```bash
sudo pacman -S stow  # arch/cachyos
sudo apt install stow  # ubuntu/debian
sudo dnf install stow  # fedora
```

### Deploy This Package

From the dotfiles directory:
```bash
cd ~/dotfiles
stow waybar
```

This creates symlinks from ~/.config/waybar/ to the configuration files in this package.

Verify deployment:
```bash
ls -la ~/.config/waybar/
# should show symlinks pointing to ~/dotfiles/waybar/.config/waybar/
```

### Deploy with Related Packages

Deploy waybar with hyprland (window manager):
```bash
cd ~/dotfiles
stow hyprland waybar
```

Or include shell configs:
```bash
cd ~/dotfiles
stow home hyprland waybar
```

### After Deployment

Restart waybar to apply any configuration changes:
```bash
killall waybar
waybar &
```

### Remove This Package

To remove the symlinks:
```bash
cd ~/dotfiles
stow -D waybar
```

## Configuration Structure

```
.config/waybar/
├── config.jsonc            # Bar layout, modules, and settings (JSONC format)
├── style.css               # Base styles
├── theme.css               # Theme-specific colors
└── modules/                # Module configurations
    ├── [individual module configs]
```

## Components

The waybar displays:
- System tray
- Clock and date
- Network status
- Audio volume
- Brightness
- Battery (on laptops)
- Workspaces
- Window title
- And other dynamic modules

## Restarting Waybar

After any configuration changes, restart waybar:

```bash
killall waybar
waybar &
```

Or use the Hyprland keybinding if configured.

## Regenerating Configuration

To regenerate this configuration from HyDE:

```bash
# If HyDE is installed
cd ~/HyDE
./Scripts/wbarconfgen.sh
```

Then copy the generated files to this package:

```bash
cp ~/.config/waybar/* ~/dotfiles/waybar/.config/waybar/
```

## Customization

After deployment to `~/.config/waybar/`, you can edit:

- `config.jsonc` - Module layout and settings
- `style.css` - Global styles
- `theme.css` - Theme colors and appearance
- `modules/*` - Individual module configurations

Changes require Waybar restart to take effect.

## Theme Changes

When the Hyprland theme changes, you may need to regenerate this configuration to match the new color scheme. See "Regenerating Configuration" section above.
